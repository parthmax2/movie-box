# Security Policy

## Reporting a Vulnerability

Please report security issues privately before opening a public issue.

Send a short report with:

- affected version or commit
- reproduction steps
- expected impact
- any relevant logs or traceback

## Scope

Security reports are welcome for:

- unsafe file writes or path traversal
- command injection
- credential or token leaks
- dependency vulnerabilities that affect normal use
- unsafe handling of downloaded metadata or filenames

Reports about third-party service availability, account access, or content
availability are outside this project's security scope.
